## 1. 目標

Sprint 11 交付了三種故事視角，但老闆看實際畫面後判定「三個視角看起來幾乎沒差異，使用者看不懂」。根因：只改了佈局和顏色，沒改「使用者看什麼」。

Sprint 12 做兩件大事：
1. **視覺風格從深色霓虹改為白紙黑格** — 全局 CSS 主題替換，白底更乾淨專業
2. **三種視角的呈現邏輯徹底重做** — 每種視角顯示完全不同的內容，不是同一堆節點換排列

**一句話驗收**：切換三種視角時，使用者反應「哇，這三個完全不同」而非「這三個有什麼不同？」

## 7. 風險

| 風險 | 等級 | 緩解方式 |
|------|------|---------|
| 全局主題替換影響範圍大 | 中高 | CSS 變數化隔離，圖稿已有完整 design tokens，機械性替換非設計探索 |
| core 目錄聚合邏輯設計 | 中 | 邏輯簡單（按 directory group），核心難點是目錄間依賴計算，可用子檔案依賴聯集 |
| 呈現邏輯重做影響現有功能 | 中高 | 基於 Sprint 11 框架改造，不是從零重寫。1092 tests 保護回歸 |
| 元件命名不改但樣式全改 | 低 | 先改樣式不改名，避免大規模重命名連鎖反應，命名重構放後續 |
| 3D 模式白底適配 | 中 | 放 P1，Three.js 材質可能需要調整，不影響核心 2D 功能 |
| dagre 在目錄級仍排成一行 | 低 | 目錄只有 5~15 個，dagre 分層更容易成功。提供回退機制 |
| 視覺回歸需人工確認 | 中 | 全局主題替換無法純靠自動測試，需 L1 逐畫面確認與圖稿一致 |

## 9. 關鍵指令：與 Sprint 11 的差異

> **⚠️ 本節為 TL 必讀。Sprint 12 不是微調，是呈現邏輯的根本改變。**

### 系統框架：從「檔案節點」到「目錄卡片」

| | Sprint 11 | Sprint 12 |
|---|-----------|-----------|
| 顯示什麼 | 所有檔案節點（50+ 個） | 目錄卡片（5~15 個） |
| 佈局效果 | dagre 排成一條水平線（失敗） | dagre 分層成功（節點少） |
| 資料來源 | 直接用檔案級 Graph JSON | core 新增 `aggregateByDirectory()` 產出目錄級數據 |
| 視覺 | Cyan 霓虹節點 | 藍色實線邊框卡片（`#1565c0` 邊框 + `#e3f2fd` 背景） |

### 邏輯運作：從「hover 高亮」到「click 聚焦」

| | Sprint 11 | Sprint 12 |
|---|-----------|-----------|
| 預設狀態 | 所有節點都可見 | 所有節點 dimmed（opacity 0.08）+ 中央提示 |
| 觸發方式 | hover 節點 | click 節點 |
| 效果 | 呼叫鏈高亮，非相關淡化至 0.15 | 呼叫鏈亮起，非相關保持 0.08（近乎消失） |
| 資訊面板 | 無 | 底部面板顯示呼叫鏈資訊 |

### 資料旅程：從「逐步亮起」到「逐步出現」

| | Sprint 11 | Sprint 12 |
|---|-----------|-----------|
| 動畫方式 | 節點逐步「高亮」（已在畫面上） | 節點逐步「出現」（從無到有） |
| 初始狀態 | 所有節點可見，逐步變亮 | 畫面空白，逐步出現節點 |
| 面板位置 | E2E 面板（現有） | 右側專用面板（同步步驟） |
| 視覺 | Green 霓虹 | 綠色實線邊框（`#2e7d32` 邊框 + `#e8f5e9` 背景） |

### 全局視覺：從「深色霓虹」到「白紙黑格」

| | Sprint 11 | Sprint 12 |
|---|-----------|-----------|
| 背景 | 深色（接近黑色） | 白紙 `#fafafa` |
| 網格 | 無 / 深色網格 | 雙層網格（20px 細線 + 100px 粗線） |
| 節點風格 | 霓虹發光 + 深色填充 | 實線邊框 + 白色填充 |
| 整體感覺 | 科技感、炫酷 | 乾淨、專業、易讀 |
