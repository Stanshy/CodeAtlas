## 1. 目標

把 8 個 Sprint 累積的功能統一在一個控制面板中，讓使用者「打開就知道能做什麼」。新增視圖模式切換（全景/依賴/資料流/呼叫鏈），一鍵看不同角度。新增端到端資料流追蹤 — 選一個入口，整條資料路線從 input 到 output 全部發亮。從「功能堆疊」進化為「一目了然」。

**一句話驗收**：打開控制面板 → 切換「資料流視圖」→ 選一個入口函式 → 端到端路線全部發亮 → 「一看就懂資料怎麼流的」。

---

## 2. 確認的流程

```
需求 → 設計 → UI 圖稿 → G1（圖稿審核）→ 實作 → G2（程式碼審查）→ 測試 → G3（測試驗收）
```

> **需要 G1** — 控制面板和視圖模式是全新 UI 模式，需老闆確認佈局和操作方式。
> **純前端 Sprint** — 不改 core/cli（除 P1 AI 設定可能需要 1 個新端點）。

### 阻斷規則

- **G1 阻斷**：控制面板 + 視圖切換的 UI 圖稿審核通過前不得開始實作

## 3. 功能清單

### P0（必做）

| # | 功能 | 描述 |
|---|------|------|
| S9-1 | 控制面板（Control Panel） | 統一側邊面板（左側可收合），整合所有功能入口和設定。區段：視圖模式、顯示偏好、分析工具、過濾器、AI 功能。取代散落各處的零散入口 |
| S9-2 | 視圖模式切換 | 四種預設視圖一鍵切換：① **全景模式** — 所有節點 + 所有邊，預設佈局 ② **依賴視圖** — 聚焦 import/export 邊，dim 其他 ③ **資料流視圖** — 聚焦 data-flow + export 邊 + symbol 標籤 + heatmap ④ **呼叫鏈視圖** — 聚焦 call 邊 + 函式節點，自動展開檔案。每種視圖為預設 filter + display 組合 |
| S9-3 | 端到端資料流追蹤 | 選一個入口節點（函式/檔案）→ 追蹤 input 如何經由 import → call → export 流到 output → 整條路徑發亮（glow effect）。面板顯示追蹤路徑的每一步 + 每步傳遞的資料（symbol / 參數 / 回傳值）。復用 Sprint 5 BFS + Sprint 7 呼叫鏈 + Sprint 8 影響分析架構 |
| S9-4 | 顯示偏好 Toggle | 控制面板中的開關：Heatmap 開/關、邊 symbol 標籤 開/關、粒子流動動畫 開/關、節點標籤密度（全顯示/智慧縮略/隱藏）、影響分析 depth limit 預設值 |
| S9-5 | Toolbar 整合修復 | ① 連接 FilterPanel Toolbar 按鈕 ② 連接 OverviewPanel Toolbar 按鈕 ③ Toolbar 佈局統一：左（控制面板 toggle）+ 中（搜尋）+ 右（視圖模式 + 2D/3D + 設定） |
| S9-6 | 2D + 3D 雙模式適配 | 控制面板、視圖模式、端到端追蹤、顯示偏好在 2D 和 3D 模式下都正常運作 |

## 7. 風險

| 風險 | 等級 | 緩解方式 |
|------|------|---------|
| 控制面板資訊過載 | 中 | 分區段收合，常用功能在上、進階設定在下。預設只展開「視圖模式」區段 |
| 視圖模式切換時狀態衝突 | 低 | 切換視圖時清除影響分析/搜尋聚焦/端到端追蹤狀態，避免混亂 |
| 端到端追蹤路徑過長 | 低 | depth limit 預設 10 + 超過 30 節點截斷 + 提示訊息 |
| 控制面板在小螢幕擋住圖 | 低 | 可收合設計 + 收合後只有 icon 列（~40px 寬） |
| Toolbar 元件過多 | 低 | 三區段佈局 + 次要功能收進控制面板，Toolbar 只放最常用 |
| 視圖預設與手動過濾疊加 | 低 | 視圖模式為「底層預設」，手動過濾疊加在上。切換視圖重置手動過濾 |
