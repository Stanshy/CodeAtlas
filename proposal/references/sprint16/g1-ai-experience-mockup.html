📥 輸入驗證 Input Validation
驗證 email 格式、密碼強度（≥8 碼、含大小寫）及用戶名稱唯一性。拒絕無效請求直接返回 400。
⚙️ 處理邏輯 Processing Logic
對密碼執行 bcrypt hash（cost=12），建立用戶 entity，呼叫 UserService.createUser 寫入資料庫，並產生 JWT token。
💾 資料操作 Data Operations
INSERT users 表，初始化 user_preferences，寫入稽核日誌。操作包於 transaction 中，失敗自動 rollback。
📤 回應格式化 Response Formatting
返回 201 + { userId, email, token }，敏感欄位（passwordHash）移除，符合 API v3 規格。
