Category 01

程式碼架構視覺化工具

業界主流的靜態程式碼結構探索工具。重點關注三視角同步佈局、 健康度色彩編碼、依賴圖的互動設計模式。

點擊放大
Sourcetrail 開源 三欄同步

三欄佈局（符號清單 + 星狀依賴圖 + 程式碼高亮），左右欄聯動：點選符號，依賴圖即時更新，程式碼捲動到對應位置。 是業界三視角同步操作的最佳範例，強調「定位 → 理解 → 閱讀」的探索動線。

對 CodeAtlas 的參考價值 三欄同步佈局是「三種故事視角」切換機制的直接對標。 可參考其左欄固定符號樹、中欄動態圖、右欄程式碼的分割比例與聯動邏輯。
點擊放大
CodeScene 商業 Circle Packing

Circle Packing 圓形填充圖 + 熱力色彩編碼：紅色代表高複雜度 / 高改動頻率的風險區域， 綠色為健康模組。面積映射檔案大小，顏色映射健康度，實現雙維度資訊密度。

對 CodeAtlas 的參考價值 可直接用於 CodeAtlas 的 Hotspot 故事視角。圓形填充作為「模組健康度總覽」的替代佈局， 比節點圖更能直覺呈現整體專案的風險分佈。
點擊放大
Dependency Cruiser 開源 靜態 SVG

Graphviz 靜態有向圖。正常依賴為灰色細線，違規依賴以紅色粗線標示， 循環依賴形成明顯閉環結構。純文字 SVG 無互動，但層次清晰，資訊密度極高。

對 CodeAtlas 的參考價值 循環依賴紅線標示語彙是架構健康視覺化的重要元素。 CodeAtlas 可沿用「問題邊加粗 + 高飽和警告色」的視覺慣例，讓問題節點一眼可辨。

Nx Graph 開源 互動式依賴圖

互動式 Web UI 深色主題依賴圖，支援 Focus 聚焦模式：選中節點後， 無關聯節點淡化至 10% 不透明度，形成強烈視覺對比，引導使用者聚焦在目標模組的上下游關係。

對 CodeAtlas 的參考價值 聚焦模式（選中 → 其餘淡化）是 CodeAtlas 端到端追蹤模式的核心互動設計參考。 建議實作時淡化非路徑節點至 15% 不透明度，路徑節點保持 100%。

Obsidian Graph View 商業 力導向圖

深色主題力導向圖，節點大小依連結數量縮放（高連結度節點更大）， hover 時鄰近節點以高亮色環顯示，無關節點不淡化（與 Nx 策略相反，保留全域感）。 整體配色最接近 CodeAtlas 現有深色霓虹風格。

對 CodeAtlas 的參考價值 力導向圖深色主題配色的直接對標，特別是節點大小映射連結度的設計原則。 Hover 高亮不淡化其他節點的策略適用於「探索模式」（vs 追蹤模式需要淡化）。

Category 02

資料流呈現方式

從 Sankey 流量圖到分散式追蹤服務地圖，研究業界如何把「資料移動的過程」視覺化， 以服務 CodeAtlas 的資料旅程故事視角。

⎸
D3 Sankey 為互動式 Observable Notebook
需訪問頁面體驗流量帶動態效果
observablehq.com/@d3/sankey ↗
D3 Sankey Diagram 開源 流量圖

資料流量帶寬度比例分配，左到右流向，節點垂直排列。流量越大的路徑帶越寬， 可直覺感受哪條呼叫路徑最頻繁。節點支援拖拽重排，帶會自動重繪貝茲曲線。

對 CodeAtlas 的參考價值 資料流故事視角的「流量分佈」子視圖。適合呈現函式呼叫的相對流量比例， 幫助識別系統瓶頸點。帶寬度 = 呼叫次數是直覺的視覺映射。

Mermaid.js 官網 Hero 區有深色主題時序圖
需訪問頁面查看互動效果
mermaid.js.org ↗
Mermaid.js 開源 時序圖

Actor 矩形 + 垂直生命線 + 水平訊息箭頭，構成標準時序圖視覺語言。 順序箭頭實線、回應箭頭虛線，活躍期用窄矩形標示在生命線上。 支援深色主題，是技術文件的事實標準。

對 CodeAtlas 的參考價值 API 呼叫鏈的「執行流程」故事視角的視覺語言直接參考。 生命線模式適合呈現非同步 await 鏈，可清楚看出哪段時間在等待。
點擊放大
Apache Airflow 開源 DAG 執行圖

DAG 流程深色主題，節點以執行狀態顏色填色：綠色=成功、紅色=失敗、 黃色=執行中、灰色=待執行。有向邊清楚標示執行先後依賴。 狀態顏色覆蓋節點背景，圖形語言一致且可快速掃描。

對 CodeAtlas 的參考價值 執行狀態顏色編碼體系可直接套用於程式碼路徑追蹤視圖。 「節點顏色 = 執行狀態」的視覺慣例是使用者已有的心智模型，降低學習成本。
點擊放大
AWS X-Ray 商業 服務地圖

圓形節點佈局：節點邊框色 = 健康度（綠/橙/紅），邊粗細 = 流量大小。 選中節點後，上下游完整路徑以高亮色保留，其餘服務淡化。 同時在右側面板顯示延遲百分位數和錯誤率分布。

對 CodeAtlas 的參考價值 路徑追蹤高亮 + 非路徑節點淡化的互動模式是 CodeAtlas 端到端追蹤的直接設計參考。 健康度色環（邊框色而非填充色）讓節點在保持辨識度的同時攜帶狀態資訊。
點擊放大
AWS X-Ray 商業 節點詳情

點選節點後，右側滑入詳細面板：延遲分佈直方圖 + P50/P90/P99 數字、 每分鐘請求數折線圖、錯誤率 / 錯誤率趨勢。 面板設計強調「從地圖下鑽到細節」的兩層操作流程。

對 CodeAtlas 的參考價值 下鑽體驗設計範本：點選圖譜節點 → 右側詳情面板滑入，不離開主視圖。 可套用於 CodeAtlas 的「點選函式節點 → 查看呼叫次數 / 影響路徑」的互動設計。

Category 03

3D 視覺化

從 Gource Git 歷史動畫到 Three.js 後處理效果，研究業界如何用 3D 空間感 和粒子特效強化視覺衝擊力，服務 CodeAtlas 的 3D 模式升級需求。

點擊放大
Gource 開源 Git 歷史 Linux Repo

Git 提交歷史動畫：目錄為樹狀分支節點，檔案為葉節點， 貢獻者以發光粒子移動至修改的節點。深黑背景 + 節點發光讓密集提交視覺震撼。 時間軸控制播放速度，可快速感受專案演化脈絡。

對 CodeAtlas 的參考價值 Sprint 11 Git 歷史故事視角的核心靈感來源。 樹狀結構 + 貢獻者粒子 + 時間軸的三元素組合是 Git 時光機模式的設計藍圖。
點擊放大
Gource 開源 高活動度 Git Repo

Git 自身 repo 的提交歷史，密集活動時形成輻射狀星爆效果。 多條活躍分支同時有貢獻者粒子移動，視覺資訊密度極高。 展示 Gource 在高頻率提交下的視覺密度處理能力。

對 CodeAtlas 的參考價值 高活動度專案的視覺密度處理策略。粒子密集時需要層次感， 建議使用深度排序（遠近虛化）或活動衰減（舊提交粒子逐漸透明化）。
點擊放大
Gource 開源 低活動度 Papervision3D

稀疏樹狀結構下的活動熱點：偶發的提交讓單一節點閃光， 長時間靜默後突然一批修改讓整個子樹爆發。 發光粒子軌跡在深黑背景下格外明顯，即使低活動度也不失優雅。

對 CodeAtlas 的參考價值 低活動度（冷門）專案的優雅呈現策略。稀疏資料下不應擠壓視覺空間， 反而應利用負空間凸顯有活動的節點，避免空洞感。
點擊放大
3d-force-graph 開源 CodeAtlas 使用中

深色背景，半透明球形節點帶內發光，細線連邊。 Three.js WebGL 渲染，支援軌道控制（旋轉/縮放/平移）。 這是 CodeAtlas 3D 模式目前使用的底層渲染庫。

對 CodeAtlas 的參考價值 CodeAtlas 3D 模式的 UI 設計基線。Sprint 11 的 3D 視覺升級需以此為出發點， 評估哪些效果可在現有 Three.js 架構上追加（Bloom Pass、粒子系統等）。
點擊放大
Three.js UnrealBloomPass 開源 後處理效果

UnrealBloomPass 泛光後處理：高亮物件（如自發光節點）的光亮向外暈散， 形成霓虹感光暈。強度、半徑、門檻值三個參數可調， 適合標記「高活躍節點」或「選中的追蹤路徑」。

對 CodeAtlas 的參考價值 3D 視角中「高活躍節點發光」的技術實作基礎。 建議對追蹤路徑上的節點啟用 Bloom，非路徑節點關閉，形成強烈的視覺引導對比。

Category 04

佈局策略對比

四種核心圖譜佈局策略的優劣對比分析。Sprint 11 三種故事視角各需要選擇最合適的佈局， 不同視角有不同的認知任務，佈局選擇直接影響使用者理解效率。

力導向佈局 (Force-Directed)

節點依物理模擬自然聚集，連結強的節點自動靠近，形成視覺上的模組群。 無固定層次，適合有機探索式瀏覽，找不認識的模組間的隱性關聯。

自然聚類 無明確層次 探索式瀏覽
代表工具：CodeAtlas（現有）、Obsidian Graph View
分層佈局 (Hierarchical Top-Down)

根節點在頂，依賴逐層向下展開，整條依賴鏈一眼掃完。 30 秒看懂整個系統框架，是「交代給新成員看」的最佳佈局。

有明確層次 快速理解 依賴方向清晰
代表工具：dagre、Graphviz dot、Sourcetrail
Sprint 11 建議：系統框架視角使用此佈局
星狀展開 (Ego Graph)

以選中節點為中心，只顯示其直接相鄰（1 度）關係。 聚焦模式，減少認知負擔，適合「我只想知道這個函式依賴誰、誰依賴它」的場景。

聚焦單一節點 1 度關係 快速定向
代表工具：Sourcetrail Focus Mode、Nx 聚焦模式
城市地圖 (City Map)

目錄 = 街區、檔案 = 建築、建築高度 = 複雜度、屋頂顏色 = 改動頻率。 提供最直覺的規模感和熱區感知，非技術人員（PM/老闆）也能快速理解。

非技術友善 規模直覺 熱區感知
代表工具：CodeScene Circle Packing、Code City、Sotomayor
Design Director

設計總監建議 — Sprint 11 視覺方向

1
系統框架視角 → 分層佈局（Hierarchical Top-Down）
採用分層佈局，參考 Sourcetrail 三欄同步佈局 + Airflow DAG 狀態色。 根節點在頂，依賴逐層向下展開，30 秒看懂整個系統架構。 左側固定目錄樹，中間 dagre 有向圖，右側程式碼預覽，三欄聯動。 節點顏色採用 Airflow 狀態色系：綠=穩定、橙=高耦合、紅=循環依賴。
2
邏輯運作視角 → 力導向佈局（保持現有）+ 路徑高亮
保持力導向佈局，參考 Obsidian hover 鄰近節點高亮策略 + AWS X-Ray 路徑追蹤淡化策略。 探索模式（無選中）：hover 時高亮鄰近節點，其餘節點不淡化，保留全域感。 追蹤模式（選中路徑）：路徑節點 100% 不透明 + Bloom 光暈，非路徑節點淡化至 15%。 兩種模式在 ViewStateContext 中切換，不重建圖形。
3
資料旅程視角 → 路徑追蹤高亮（Path Tracing）
採用路徑追蹤高亮視覺語言，參考 Datadog / AWS X-Ray 的 fade-out 策略 + Gource 粒子軌跡效果。 選定入口函式後，端到端路徑高亮（節點 + 邊），動畫粒子沿路徑流動表示資料流向。 非路徑節點淡化，右側面板顯示路徑上每個節點的延遲 / 呼叫次數。 粒子流速映射實際呼叫頻率（高頻 = 粒子更密集、更快）。

整體風格：繼續深色霓虹主題（bg #0a0a0f），不改變既有視覺識別。 3D 模式考慮追加 Three.js UnrealBloomPass 泛光後處理效果，對高活躍節點和追蹤路徑啟用 Bloom， 增強視覺衝擊力，同時以 Gource 的粒子軌跡設計作為 Git 歷史故事視角的核心動畫語彙。 建議以 3d-force-graph 現有架構為基線，評估 Bloom + 粒子系統的效能預算再決定實作範圍。
