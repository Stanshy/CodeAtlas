## 1. 目標

讓使用者從「看」升級到「預測」。選一個節點，一鍵看到所有下游影響或上游依賴。搜尋後整張圖聚焦到相關路徑。AI 一鍵生成專案架構概述。從「分析工具」變成「決策工具」。

**一句話驗收**：右鍵節點 →「顯示影響範圍」→ 下游全亮 → 搜尋聚焦 → AI 一鍵概述 → 「這工具真的能幫我做決策」。

## 3. 功能清單

### P0（必做）

| # | 功能 | 描述 |
|---|------|------|
| S8-1 | 正向影響分析 | 選節點 → BFS 遍歷所有下游（被依賴者）→ 高亮整條影響鏈。支援 depth limit（預設 5）。面板顯示影響節點列表 + 影響層數 |
| S8-2 | 反向依賴分析 | 選節點 → BFS 遍歷所有上游（依賴者）→ 高亮整條依賴鏈。與正向共用演算法（反轉方向） |
| S8-3 | 搜尋聚焦模式 | 搜尋後整張圖暗掉，只有匹配節點 + 其直接依賴路徑亮起。Escape 退出聚焦模式。強化現有 SearchBar |
| S8-4 | 模組過濾 | FilterPanel：按目錄（checkbox tree）、節點類型（file/directory/function/class）、邊類型（import/call/data-flow）過濾。過濾後圖只顯示符合條件的節點和邊 |
| S8-5 | AI 專案概述 | 一鍵生成整個專案的架構摘要。遍歷全圖結構（節點數、模組關係、核心模組）→ 組裝 prompt → AI 生成 1-2 段概述。只送結構資訊不送原始碼（保護隱私）。新增 POST /api/ai/overview 端點 |
| S8-6 | 2D + 3D 雙模式適配 | 影響分析高亮、搜尋聚焦、過濾在 2D 和 3D 模式下都正常運作 |

### P1（應做，視時間）

| # | 功能 | 描述 |
|---|------|------|
| S8-7 | 自然語言搜尋 | 輸入自然語言描述（「處理登入的邏輯」）→ AI 提取關鍵字 → 匹配節點 label/path/AI 摘要 → 聚焦到相關節點 |

## 4. 範圍界定

### 做

- 正向影響分析（下游 BFS + depth limit）
- 反向依賴分析（上游 BFS）
- 搜尋聚焦模式（暗掉無關 + 亮起相關）
- 模組過濾面板（目錄/類型/邊類型）
- AI 專案概述（結構摘要，不送原始碼）
- 2D + 3D 適配
- 自然語言搜尋（P1）

### 不做

- 即時影響分析（檔案 watch → 自動更新）
- Git diff 視覺化（Phase 5）
- AI 程式碼生成/建議
- 影響分析量化（風險分數）
- 跨專案依賴分析

## 7. 風險

| 風險 | 等級 | 緩解方式 |
|------|------|---------|
| 大型專案影響範圍過大 | 中 | depth limit 預設 5 + 可調整 + 超過 50 節點提示「影響範圍較大，已截斷」 |
| AI 概述 prompt 過長 | 低 | 只送結構（節點名/類型/邊數），不送原始碼。超過 token limit 摘要化 |
| FilterPanel 狀態管理 | 低 | 擴充 ViewStateContext，與現有過濾邏輯分離 |
| 自然語言搜尋品質 | 低 | P1，品質取決於 AI 模型。先做關鍵字提取再匹配，fallback 為精確匹配 |
| 右鍵選單在 3D 模式 | 低 | 3D 模式用 raycaster 偵測節點，彈出 DOM overlay 選單 |
